# Agile Story Writing Checklist

The following list is intended to help with the development of comprehensive Stories and Acceptance Criteria

:::note

Not all of these components will be considerations for every story. 

:::

* Identify new functionality vs existing
* Error/Success Messaging provided and conditions explained
* Page Titles/Navigation match
* Page Context labeling provided
* Post-backs visual cues or messaging (when they are needed for filtering) 
* Column labels are consistent and logical (consistency for data throughout application)
* Links are used for navigation/ buttons are used for data
* Links and Buttons are labeled consistently throughout application
* Use Case explanations with steps or process flows are provided
* Business rules/Dependencies/Exceptions are called out
* Statuses identified and state chart defined
* Table columns – Describe data displayed and  Identify if nulls are allowed 
* Table filters – defaults, how drop-downs are derived, required or not
* Dialog fields – required?/ default data
* Possible Regression issues
* Backward compatibility
* Potential security concerns
* Potential performance concerns
* Documentation/Training needs
